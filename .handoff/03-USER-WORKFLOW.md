# 03 — Как работать в Antigravity (для нулевика)

> Цель: научить тебя ставить задачи в Antigravity так же легко как ты делал это с Devin. **Минимум технических знаний, максимум качества.**

## Главное различие Antigravity vs Devin

| | Devin (web) | Antigravity (desktop) |
|---|---|---|
| Где работает | Браузер | На твоём компе (Mac/Win/Linux) |
| Видно процесс | Только ответы | Видишь код, файлы, терминал, всё |
| Контроль | Просто ждёшь | Можешь править руками если хочешь |
| Цена | Платные ACU | Бесплатная квота на топ-модели |
| Скорость | Чуть медленнее (запуск VM) | Мгновенно — он на твоём компе |
| Превью | Vercel preview | Vercel preview (то же) |

**Главное правило:** **не открывай Editor если ты ничего там не понимаешь.** Работай только в **Agent Manager** — это аналог Devin-чата. Туда пиши задачи, оттуда получай результаты.

## Антигравити-интерфейс — что где

```
┌──────────────────────────────────────────────────────────────┐
│  Agent Manager (Cmd+E / Ctrl+E чтобы переключить в Editor)   │
├──────────┬───────────────────────────────────────────────────┤
│          │                                                   │
│  Список  │        Активный агент / workspace                 │
│  агентов │                                                   │
│          │   [Чат]          [Файлы]      [Терминал]          │
│ - Agent1 │   ───────         ──────       ─────────          │
│ - Agent2 │   Твои промпты                                    │
│ - Agent3 │   Ответы агента                                   │
│   ...    │                                                   │
│          │                                                   │
│ + New    │   ┌──────────────────────────────────┐            │
│   Task   │   │ Напиши задачу здесь...          │            │
│          │   └──────────────────────────────────┘            │
└──────────┴───────────────────────────────────────────────────┘
```

Левая колонка — список твоих агентов (как в Devin sessions). Можешь иметь несколько параллельно.

## Workflow для каждой задачи — шаблон

1. **Создай нового агента** (`+ New Task` в левой колонке)
2. **Выбери workspace** — папку с репо `ice-logix-frontend`
3. **Выбери модель** — по умолчанию Gemini 3 Pro (премиум, бесплатно). Если квота закончилась — Gemini 3 Flash
4. **Напиши промпт** (см. шаблоны в [`04-PROMPT-TEMPLATES.md`](./04-PROMPT-TEMPLATES.md))
5. **Жди** — агент строит план, шагает по шагам, сам коммитит и пушит
6. **Проверь PR** — кликни на ссылку которую он выдал, открой Vercel Preview
7. **Если ОК** — мержь PR на GitHub (или попроси агента: «смержи PR»)
8. **Если не ОК** — пиши прямо в чат: «вот тут не работает X, посмотри»

## Что нужно знать о промптах

### Хорошо vs плохо

❌ **Плохо:** «Сделай легит-чек»
- Слишком абстрактно. Агент не знает что делать первым: UI? backend? БД?

✅ **Хорошо:** «Сделай Edge Function `legit-check` в `supabase/functions/legit-check/index.ts`. Принимает `{photo_base64, brand_hint?}`, вызывает Gemini Vision из OpenRouter, сравнивает с auth_markers из `legit_check_models`, возвращает `{score, issues[], confidence}`. Логику смотри в `01-PROJECT-CHEATSHEET.md` секция 15.»

### Структура хорошего промпта

```
[ЧТО] Краткое описание задачи
[ГДЕ] Где живёт код (файл/папка)
[КАК] Логика / алгоритм / pseudocode
[КОНТЕКСТ] Ссылка на cheatsheet секцию или предыдущие PR
[РЕЗУЛЬТАТ] Что я ожидаю (PR? Тест? Файл?)
```

Не обязательно следовать ему буквально. Если задача простая — пиши свободно. Если сложная — структурируй.

### Использование `@`-меншенов

В чате Antigravity ты можешь писать `@`-меншенами:
- `@files index.html` — агент сразу читает этот файл
- `@symbol enhanceQuery` — агент сразу читает эту функцию
- `@docs Tailwind` — агент сразу подтягивает доки через Context7

Это супер экономит токены и время.

## Как обращаться к Claude Code из Antigravity

Когда задача СЛОЖНАЯ (большой рефакторинг, архитектура), переходи на Claude Code:

1. В Antigravity открой Terminal (`Ctrl+` `)
2. Введи:
   ```bash
   claude "опиши задачу одним предложением"
   ```
3. Claude Code запустится в чат-режиме внутри терминала, прочитает репо, спросит уточнения если надо
4. По окончании — он сам предложит запушить или скажет тебе сделать `git commit`

**Когда использовать Claude Code:**
- Сложный рефакторинг (≥5 файлов)
- Архитектурные решения («как лучше структурировать X»)
- Если Gemini 3 Pro даёт некачественный код 2 раза подряд

**Когда НЕ использовать Claude Code:**
- Простые баги (один файл, известная проблема) — это для Antigravity-агента
- UI-правки (CSS, текст, цвета) — Antigravity справляется
- Документация / комменты — Antigravity справляется

## Дизайн-помощник: v0.dev workflow

Когда тебе нужен новый UI:

1. Открой <https://v0.dev>
2. Опиши что хочешь (см. готовый промпт в `02-SETUP-GUIDE.md` шаг 5)
3. Когда v0 покажет вариант — скопируй код (`Copy code`)
4. В Antigravity создай агента:
   ```
   Я сгенерировал UI через v0.dev. Адаптируй его под наш `index.html` (vanilla JS + Tailwind CDN, без React). Замени React JSX на DOM-методы. Стиль и логика должны соответствовать существующим компонентам.
   
   Вот код от v0:
   
   ```jsx
   <вставь сюда код от v0>
   ```
   
   Куда вставить: новый экран `screenLegitCheck` после `screenCalculator`. Кнопка перехода — в Calculator под кнопкой «Сохранить заказ».
   ```

Antigravity сам разберётся как сконвертить React в vanilla JS.

## Параллельная работа

В Antigravity ты можешь запускать **несколько агентов одновременно**. Каждый работает в своём workspace.

Пример:
- Agent 1: «Реализуй PR-B (Edge Function legit-check)»
- Agent 2: «Реализуй PR-C (UI кнопка AI Проверка)»
- Agent 3: «Пофикси Bug #1 (навигационные лаги)»

Все три могут идти параллельно. **НО:** если они правят один файл — будут конфликты при мердже. Лучше держать максимум 2 параллельных агента и на разных областях кода.

## Что делать когда агент тупит

1. **Скажи ему остановиться:** «остановись, проблема в X, не идёт по правильному пути»
2. **Перезапусти задачу** с более конкретным промптом
3. **Поменяй модель** на Claude Code если задача архитектурная
4. **Спроси меня вопрос** через cheatsheet — там 90% ответов

## Что делать когда CI красный

В Antigravity ты увидишь PR-checks статус прямо в чате (если установлен GitHub MCP). Если что-то упало:

1. Скажи агенту: «CI failed, посмотри логи и пофикси»
2. Агент сам подтянет логи через GitHub MCP
3. Если 3 итерации не помогли — переключайся на Claude Code

## Окончательные правила экономии (для качества)

1. **Один агент = одна логическая фича.** Не пихай в один промпт «сделай 5 разных вещей».
2. **Используй промпт-шаблоны** из `04-PROMPT-TEMPLATES.md`.
3. **Контекст-файлы** (`01-PROJECT-CHEATSHEET.md`) — закидывай как `@file` если агент тупит.
4. **Превью-режим** — не сразу пушь. Сначала посмотри Diff View в Antigravity.
5. **Не правь код руками** если не уверен. Лучше попроси агента поправить.
